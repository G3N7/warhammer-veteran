# Session Retrospective Workflow

## Purpose
Analyzes completed user sessions to extract learnings, identify workflow improvements, and discover user preferences. Meta-workflow for continuous module improvement.

## Trigger
Can be triggered by any agent after significant interactions, or manually by user request.

## Outputs
- User preference discoveries
- Workflow pain points identified
- Requirement suggestions
- Agent memory updates

## When to Use
- After complex multi-agent workflows
- When user requests requirement extraction
- Periodic review of agent performance
- Module improvement cycles
